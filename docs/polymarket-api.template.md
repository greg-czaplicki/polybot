## Purpose
This file is the operational API reference for this repo. Keep only endpoints and fields we actually use.

## Critical Gotchas
- `GET /markets` uses `condition_ids` (array query param), not `conditionId`.
- `GET /markets/{id}` expects Gamma market `id`, not a condition id.
- Settlement logic should require exact id match before writing outcomes.

## Endpoint Map
| Area | Endpoint | Query/Path Key | Primary Fields | Used By |
|---|---|---|---|---|
| Gamma markets | `GET /markets` | `condition_ids` | `id`, `conditionId`, `closed`, `umaResolutionStatus`, `outcomes`, `outcomePrices` | Auto-settle |
| Gamma market detail | `GET /markets/{id}` | `id` (path) | `closed`, `umaResolutionStatus`, `outcomes`, `outcomePrices` | Future settle path |

## Settlement Rules
1. Lookup market by `condition_ids` or (preferably) stored `gammaMarketId`.
2. Require exact match to target identifier.
3. Settle only when resolution status is definitive.
4. Persist `status`, `resolved_outcome`, `close_price`, `roi`, `clv`, `settled_at`.

## Verification Curls
```bash
# 1) Condition-id lookup (spec-correct)
curl -s "https://gamma-api.polymarket.com/markets?condition_ids=<CONDITION_ID>&limit=50" \
| jq --arg cid "<CONDITION_ID>" '[.[] | select(.conditionId==$cid)] | .[0] | {id,conditionId,closed,umaResolutionStatus,outcomes,outcomePrices}'

# 2) Market-id lookup
curl -s "https://gamma-api.polymarket.com/markets/<MARKET_ID>" \
| jq '{id,conditionId,closed,umaResolutionStatus,outcomes,outcomePrices}'
```
